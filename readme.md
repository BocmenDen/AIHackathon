# **AIHackathon** - Чат-бот для оценки моделей прогнозирования данных на Хакатоне по ИИ

Этот проект представляет собой Telegram-бота, созданного с использованием библиотеки [OneBot](https://github.com/BocmenDen/OneBot), предназначенного для автоматической оценки качества моделей прогнозирования данных в рамках хакатона по искусственному интеллекту.

## Функциональность

Бот позволяет участникам хакатона загружать свои модели прогнозирования и автоматически оценивать их качество на заданных данных.  Оценка осуществляется с помощью отдельного Python-скрипта [GetAccuracy.py](https://github.com/BocmenDen/AIHackathon/blob/main/AIHackathon/Raw/GetAccuracy.py), который запускается как отдельный процесс.  Скрипт принимает на вход два параметра: путь к модели и путь к файлу с данными.

Бот поддерживает две роли:

* Участник: Загружает модели для оценки. При первом запуске бот выдает уникальный идентификатор пользователя. Этот ID необходим для связи профиля пользователя в боте с данными, полученными из внешних источников (например, онлайн-форм).
* Администратор:  Объединяет участников в команды, предоставляет доступ к боту и добавляет метаданные пользователей, полученные из внешних источников (например, онлайн-форм).  Администратор загружает данные в формате CSV-файла с тремя столбцами: 
	* уникальный ID пользователя;
	* ФИО пользователя;
	* название команды (должны точно совпадать для всех участников одной команды).

Вот пример CSV-файла. Важно: в качестве разделителя используется точка с запятой «;».
```csv
1;Иванов Иван Иванович;Команда А
2;Петрова Анна Сергеевна;Команда А
3;Сидоров Петр Алексеевич;Команда B
4;Кузнецова Мария Дмитриевна;Команда B
```

## Развертывание

Для запуска приложения необходимо указать параметры конфигурации, а в качестве входных данных Main указать путь к файлу конфигурации:
```json
{
  "bot_helpInfoPath": "./Raw/HelpInfo.txt",
  "pythonNameFunction": "getAccuracyScore",
  "pythonPathScript": "./Raw/GetMetrics.py",
  "pythonDBTarget": "СТОЛБЕЦ ДЛЯ КОТОРОГО СТРОЯТСЯ PREDICT",
  "pythonPathExe": "python",
  "bot_commandKeyboard": "keyboard",
  "bot_linkSurvey": "ССЫЛКА НА ОПРОС-РЕГИСТРАЦИЮ",
  "bot_isFilterUsers":  false,

  "tg_token": "ТОКЕН TELEGRAM БОТА",
  "DBPath": "ПУТЬ КУДА СОХРАНЯТЬ БД",
  "pythonPathDBModel": "ПУТЬ К ФАЙЛУ С ДАННЫМИ ДЛЯ ПРОГНОЗИРОВАНИЯ",
  "modelsPathStorage": "ПУТЬ К ПАПКЕ КУДА СОХРАНЯТЬ МОДЕЛИ УЧАСТНИКОВ"

  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.EntityFrameworkCore": "None"
    }
  }
}
```
Замените:

* "tg_token" на ваш токен Telegram-бота.
* "DBPath" на путь к базе данных.
* "pythonPathDBModel" на путь к файлу с данными для прогнозирования.
* "modelsPathStorage" на путь к папке для хранения моделей участников.
* "bot_isFilterUsers" позволяет игнорировать пользователей не зарегестрированных в системе.
* "Logging" позволяет выключить логирование Entity Framework.


## Управление пользователями

* Уникальный ID пользователя:  Выдается ботом при первом запуске.  Этот ID используется для связи данных пользователя из бота с данными из внешних источников.
* Администраторы:  Для предоставления административных прав необходимо использовать внешний инструмент для изменения значения поля IsAdmin на true в базе данных, используя уникальный ID пользователя.
